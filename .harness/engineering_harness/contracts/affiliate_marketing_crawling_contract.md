# Engineering Contract: Affiliate Marketing, Crawling & Product Ingestion Module

This contract defines the specifications, deliverables, and verification metrics required to implement the Affiliate Marketing and Automated Web Ingestion Module for Seliabot, specifically tailored for the Ibera Club use case.

---

## 1. Scope & Deliverables

The deliverables for this module are:

### 1.1. Automated Deep Scraping & Product Ingestion API
- Create a new endpoint `POST /api/products/scrape-import` in [platform-api/src/api/routes/products.ts](platform-api/src/api/routes/products.ts).
- This route will accept a `seedUrl` and optionally `maxPages` and `maxDepth` parameters.
- It will execute the crawler from [platform-api/src/services/crawlerService.ts](platform-api/src/services/crawlerService.ts) and the scraper from [platform-api/src/services/scraperService.ts](platform-api/src/services/scraperService.ts).
- The scraped body texts will be sent to Gemini via the Google Gen AI SDK (mirroring the parser in `/api/products/extract-doc`) using a tailored prompt that structures the unstructured text into clean records conforming to our product database schema.
- Extracted products are to be upserted in the database via [platform-api/src/db/repositories/productRepo.ts](platform-api/src/db/repositories/productRepo.ts) under the active tenant.

### 1.2. Database & Schema Upgrades
- Add `affiliate_id` (VARCHAR) and `affiliate_slug` (VARCHAR) fields to the `agent_configs` and `tenants` tables (or `tenant_settings`) to represent the referral tracking metadata.
- Add `promo_start_date` (TIMESTAMP) and `promo_end_date` (TIMESTAMP) fields to the `products` table to support automatic promotional scheduling.
- Extend [platform-api/src/types/index.ts](platform-api/src/types/index.ts) to define these new attributes in types `Product`, `CreateProductInput`, and `AgentConfig`.

### 1.3. Dynamic Affiliate Link Rewriting
- Integrate a link-rewriter helper within the conversation loop in [platform-api/src/agent/agentController.ts](platform-api/src/agent/agentController.ts).
- When sending a product card or purchase link to a B2C client, the system will look up the agent config's `affiliate_id` or `affiliate_slug`.
- If present, Seliabot will automatically parse the outgoing product link (e.g. `https://ybera.pa/product-name`) and append the referral tag (e.g. `https://ybera.pa/product-name?ref=8023963` or `https://iberaclub.pa/YoleliAndrade/product-name`) based on tenant-specific routing rules.

### 1.4. In-App Promotional Scheduling
- Update [platform-api/src/db/repositories/productRepo.ts](platform-api/src/db/repositories/productRepo.ts) query logic so that whenever a product's price is requested, if `CURRENT_TIMESTAMP` is within the product's `promo_start_date` and `promo_end_date`, the system applies the promotional discount automatically instead of the standard retail price.

### 1.5. Dashboard "Import from Website" UI Component
- In [platform-dashboard/src/pages/Products.tsx](platform-dashboard/src/pages/Products.tsx), add an "Import from Website" button in the action bar.
- Clicking the button will open an interactive modal that prompts the user for a Website URL, calls the `POST /api/products/scrape-import` API, displays a table of extracted products (with images, prices, categories), and lets the user click "Approve and Save" to insert them into their active Seliabot catalog.

---

## 2. Verification Metrics (Definition of Done)

- **Verification of Web Ingestion**: Passing a product page URL successfully parses, structures, and registers the products inside the PostgreSQL DB under the tenant catalog.
- **Verification of Referral Attribution**: Outbound links generated by the AI assistant in mock chat threads correctly append the active affiliate referral string.
- **Verification of Promotion Validity**: Product searches outside the promotion validity dates return the standard price, while searches within the range return the discounted promotional price.
- **Verification of Build Stability**: Running compiling and typecheck checks on both `platform-api` and `platform-dashboard` passes with 100% success.
